气泡渲染探针 · Bubble Render Probe

本页零登录、纯本地，是辅助/佐证信号，不是可行性的决定性证据。决定性证据是 adb shell dumpsys activity activities：气泡展开后 resumed 的顶层 Activity 必须是目标浏览器包的 Custom Tab Activity、且处在气泡所在 task/window 内（详见 spec）。本页只做两件事：给出引擎的辅助判断，以及测同源一方存储（cookie / localStorage）是否与独立打开的同一浏览器连续。对同一 URL 分别用「独立浏览器直开」「原生气泡」各开一次截图对比。

1) 引擎辅助信号（非决定性）：User-Agent 是否含 wv
完整 UA（以此为准人工判读，UA 可被改写，仅辅助）
2) 会话/存储共享令牌（cookie 与 localStorage 各自独立生成、独立比较）
cookie 令牌（含写入复读校验）
localStorage 令牌（与 cookie 令牌互不 seed）
localStorage 首访时间（独立第三信号）
3) 判读方法
  • 引擎（决定性证据在 dumpsys，本页仅辅助）：气泡内 UA 含 wv ⇒ 强烈提示内置 WebView；不含 wv 只是不排除外部 CCT，不能单独据此判成功——须配合 dumpsys 的包名/栈证据。
  • cookie 共享：独立浏览器直开生成 cookie 令牌 X（须显示"写入已复读校验 OK"）。气泡内打开同一 URL：cookie 令牌= X ⇒ 与该浏览器共享 cookie jar；≠ X 或校验失败 ⇒ 未共享 / 无效，本信号作废不参与判定。
  • localStorage 共享：独立比较 localStorage 令牌与首访时间是否与独立浏览器一致（不用 cookie 令牌回填，避免假阳）。cookie 共享但 localStorage 不共享是可能的，须分别记录。
